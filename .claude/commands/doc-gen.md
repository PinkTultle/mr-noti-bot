$ARGUMENTS로 지정된 파일 또는 최근 변경된 파일에 대해 코드 설명 문서를 생성/갱신한다.

## 수행 절차

1. **대상 파일 결정**
   - 인자가 있으면 해당 파일
   - 인자가 없으면 `git diff --name-only HEAD~1`로 최근 변경 파일

2. **문서 생성**
   - 각 소스 파일에 대해 `docs/source/` 하위에 동일 경로로 `.md` 파일 생성
   - 예: `src/drivers/uart.c` → `docs/source/src/drivers/uart.md`

3. **문서 내용**
   ```markdown
   # {파일명}

   ## 목적
   (이 모듈이 하는 일)

   ## 주요 함수/클래스
   | 이름 | 역할 | 매개변수 |
   |------|------|----------|

   ## 의존 관계
   (include/import 분석)

   ## 주의사항
   (알려진 제한, 특이사항)
   ```

4. **기존 문서 갱신**
   - 이미 존재하면 변경된 부분만 업데이트
   - "목적"과 "주의사항"은 기존 내용 보존 (수동 작성 존중)

## 규칙

- 코드 주석과 중복하지 않는다 — 주석은 "왜", 문서는 "전체 그림"
- 자동 생성 마커 `<!-- auto-generated -->` 를 파일 상단에 포함
