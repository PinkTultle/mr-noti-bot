$ARGUMENTS로 지정된 파일 또는 스테이징된 변경사항을 코드 리뷰한다.

## 수행 절차

1. **대상 결정**
   - 인자가 있으면 해당 파일
   - 인자가 없으면 `git diff --cached` (스테이징된 변경)
   - 스테이징도 없으면 `git diff` (워킹 트리 변경)

2. **리뷰 체크리스트**
   - 보안: 인젝션, 하드코딩 시크릿, 버퍼 오버플로우
   - 성능: 불필요한 루프, 메모리 누수, 블로킹 호출
   - 가독성: 네이밍, 함수 길이, 복잡도
   - 규칙 준수: CLAUDE.md 및 rules/ 규칙 위반 여부

3. **출력 형식**
   ```
   ## 코드 리뷰 결과

   ### 심각도별 요약
   | 심각도 | 건수 |
   |--------|------|
   | 🔴 Critical | N |
   | 🟡 Warning  | N |
   | 🟢 Info      | N |

   ### 상세 피드백
   #### [심각도] 파일:라인 — 제목
   설명 + 수정 제안
   ```

## 규칙

- 코드를 수정하지 않는다 — 리뷰 + 피드백만
- 프로젝트 `.claude/rules/`의 코딩 표준을 기준으로 판단
