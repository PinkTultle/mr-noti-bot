이번 세션의 작업 내용을 정리하여 HANDOFF.md를 갱신하고, 커밋 후 푸시한다.

## 수행 절차

1. **현재 상태 수집**
   - `git log --oneline -20`으로 이번 세션 커밋 파악
   - `git diff --stat HEAD~5`로 변경 파일 요약
   - `VERSION` 파일 확인

2. **HANDOFF.md 갱신**
   - "마지막 작업 요약" 섹션을 이번 세션 내용으로 교체
   - 날짜: 오늘 날짜
   - 이번 세션 완료 작업: 커밋 기반으로 정리
   - 현재 상태: 버전, 브랜치, 스킬/에이전트/rules 수
   - 주의사항: 미완료 작업, 알려진 이슈
   - 다음 작업 후보: 우선순위 순으로 정리
   - 저장소 구조: 변경이 있으면 갱신

3. **커밋 + 푸시**
   - `docs: HANDOFF.md 갱신 — <한 줄 요약>`
   - `git push origin main`

## 규칙

- 기존 HANDOFF.md 형식과 섹션 구조를 유지한다
- 커밋 메시지에서 작업 내용을 추론하되, 대화 맥락도 참고한다
- "워크스테이션 환경 메모" 섹션은 변경하지 않는다 (defconfig이 관리)
