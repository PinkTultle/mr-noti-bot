- **작업 개요**: main.go를 리팩토링하여 executeJob과 summaryJob 2개를 같은 go-quartz 스케줄러에 등록하고, one-shot 판정 로직을 summary.schedule까지 확장

| # | 항목 | 상태 | 비고 |
|---|------|------|------|

## 설계 대비 변경

- Design 섹션 4.2에서는 `mainStandalone` 내부에 직접 조건문을 두었으나, 테스트 용이성과 가독성을 위해 `shouldRunOneShot(config *Config) bool` 헬퍼로 추출했다. 동작은 설계와 동일하다.
- `runScheduler` 시작 시점의 `log.Printf("Running in cron mode with schedule: %s", ...)` 로그를 제거했다. 각 job 등록 시 개별적으로 schedule을 출력하므로 중복을 피했다. Design 섹션 4.1 참조 구현과 일치.

## 기술 부채

없음.

의존: `executeSummary(config *Config) error`는 summary-dev가 summary.go에 구현 중이며, `SummaryConfig`/`Config.Summary` 필드는 config-dev가 이미 config.go에 추가 완료 (Task #1 completed). main.go 단독 컴파일은 summary-dev 완료 후 검증 가능.
