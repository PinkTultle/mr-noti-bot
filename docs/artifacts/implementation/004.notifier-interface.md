## 설계 대비 변경

### 1. WebhookNotifier.Send 테스트 방식

설계에서는 WebhookNotifier의 webhook 호출을 직접 검증하도록 하지만, `slack.PostWebhook`은 패키지 레벨 함수로 mock이 불가능하다. 대신 그룹핑 로직과 포맷 로직을 별도 함수로 분리하여 해당 함수들을 단위 테스트한다. WebhookNotifier.Send의 빈 입력 케이스는 직접 검증 가능.

### 2. DMNotifier.client 필드 타입

설계 원문에서는 `client` 필드를 "nil by default, created on first Send"로 명시한다. Task 명세를 그대로 따라 `SlackAPI` 인터페이스 타입으로 선언하여 테스트 시 mock 주입 가능하게 구현했다.

### 3. Unicode 문자 사용

설계의 화살표 `->` 와 불릿 `*`을 Slack 렌더링 호환을 위해 유니코드로 변환:
- `->` -> `\u2192` (right arrow)
- `*` -> `\u2022` (bullet)
