## 설계 대비 변경

- Design 3.4의 의사코드에 등장한 `splitSegmentsByMRCount` 별도 함수는 도입하지 않고
  `formatSummaryMessages` 내부 패킹 루프로 통합했다. 구현 단순화 + 테스트 대상 축소.
  분할 요구사항(상태 섹션 단위 유지, 단일 상태 40 초과 시 재분할)은 동일하게 충족.
- `formatSummaryMessages`는 40개를 초과한 단일 상태 그룹을 내부에서 40개 단위로 먼저
  잘라낸 뒤 패킹한다. 각 조각은 동일한 상태 헤더로 반복 출력된다.
- 분할된 모든 메시지에 최상단 `:clipboard: *MR 현황 요약*` 헤더와 총계를 동일하게
  포함하도록 했다 (테스트 요구: `TestFormatSummaryMessages_SplitByCount`).

## 기술 부채

없음. 모든 헬퍼/포맷 함수가 nil-safe. Author나 CreatedAt이 nil인 엣지 케이스에서도
방어 코드로 crash 없이 동작.
