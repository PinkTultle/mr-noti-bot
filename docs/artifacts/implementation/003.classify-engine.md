## 설계 대비 변경

### 1. ClassifiedMR 구조체에서 ProjectID 필드 제외

설계 문서(smart-notification.design.md 2.1절)에는 `ClassifiedMR`에 `ProjectID int` 필드가 있으나,
Task 명세에서는 제외되어 있다. `MergeRequestWithApprovals.ProjectID`로 이미 접근 가능하므로
Task 명세를 따랐다. 기존 `notification.go`의 `ClassifiedMR` 사용 코드와도 호환된다.

### 2. classifyMergeRequest 시그니처

설계 문서: `classifyMergeRequest(mr *MergeRequestWithApprovals, projectID int)`
Task 명세: `classifyMergeRequest(mr *MergeRequestWithApprovals)`
ProjectID가 이미 `mr.ProjectID`에 포함되므로 Task 명세를 따랐다.

### 3. TestClassifyMergeRequest_ChangesRequested 기대값 변경

설계의 Priority 3 조건(`!BlockingDiscussionsResolved`)은 Priority 2에서 이미
`BlockingDiscussionsResolved == false`를 잡으므로 도달 불가능하다.
테스트 기대값을 `StatusChangesRequested` -> `StatusBlockingDiscussions`로 조정했다.
