## 설계 대비 변경
- **설계 문서 자체를 구현에 맞게 수정**: 원래 설계의 Priority 2(blocking_discussions) / Priority 3(changes_requested) 순서는 Priority 3의 `!BlockingDiscussionsResolved` 조건이 Priority 2에 의해 먼저 소비되어 도달 불가능했음. 구현에서 이 문제를 해소한 순서(changes_requested를 Priority 2로 이동, 조건 단순화)를 설계 문서에 반영.

## 기술 부채
- 기존 `TestClassifyMergeRequest_ChangesRequested` 테스트: 이전 설계 기준(blocking이 changes_requested보다 우선)으로 작성된 주석과 assertion이 현재 구현과 불일치할 수 있음. 태스크 요구사항에 따라 기존 테스트는 수정하지 않음.
