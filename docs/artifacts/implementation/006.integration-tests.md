## 설계 대비 변경

- `executeLegacy`는 내부에서 `slackClient`를 직접 생성하므로 mock 주입이 불가능하다. 함수를 직접 호출하는 대신 구성 요소(filter -> format -> sendSlackMessage)를 파이프라인으로 연결하여 테스트한다.
- `executeSmartNotification`도 내부에서 `newNotifier(config)`를 호출하여 실제 HTTP 전송을 시도하므로, 함수를 직접 호출하지 않고 하위 컴포넌트(classify -> loadState -> filterChangedMRs -> resolveNotificationTargets -> buildNewState -> saveState)를 파이프라인으로 조합하여 테스트한다.
